# 上下文总结

上下文总结功能可以在对话上下文过长时自动生成总结，节省Token消耗。当上下文使用超过设定比例或消息条数达到阈值时，系统会自动触发总结。

## 使用方法

1. **进入设置**：在设置页面找到"上下文与总结"选项，点击进入。

![设置页面](/manuals/assets/chat/summary/44ea766adae93ca02bf620a586ff24d6.jpg)

2. **配置上下文长度**：
   - **上下文长度**：设置模型记忆对话长度（建议设为日常使用的最大值）
   - **最大上下文长度**：设置在Max模式下使用的上下文长度（建议设为模型API支持的最大值）

3. **启用对话总结**：打开"启用对话总结"开关，系统会在上下文过长时自动生成总结。

4. **设置触发条件**：
   - **上下文使用比例阈值**：当上下文使用超过此比例时触发总结（范围0.1~0.95，默认0.7）
   - **按消息条数触发总结**：开启后，当新消息数量达到阈值时触发总结
   - **消息条数阈值**：设置触发总结的消息条数（默认8条）

![上下文和总结设置](/manuals/assets/chat/summary/11b21357a90214d5c6637d51707657d1.jpg)

## Max模式

Max模式使用更大的上下文窗口，适合处理长对话或需要更多上下文信息的场景。

### 使用方法

在对话设置菜单中，打开"Max模式"开关即可启用。

![Max模式开关](/manuals/assets/chat/summary/89624e415e2ac0ed7a059a15d8bcd043.jpg)

## 使用场景

- **长对话管理**：在长时间对话中，自动总结历史内容，保持对话流畅
- **节省Token**：通过总结压缩上下文，减少API调用成本
- **处理复杂任务**：使用Max模式处理需要大量上下文信息的复杂任务
